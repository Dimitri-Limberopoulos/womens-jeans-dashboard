BUILD COMPONENTS (For Reference)

prep_data.py (12 KB)
  - Python 3 script to clean raw Excel data
  - Parses prices, fit descriptions, rise categories
  - Maps 500+ colors to 6 wash categories
  - Extracts cotton percentages
  - Outputs data.json with cleaned + aggregated data
  
  Usage: python3 prep_data.py

app.js (16 KB)
  - ES5-only JavaScript (no arrow functions, no template literals)
  - Dashboard application logic
  - Chart.js integration
  - Brand filtering
  - Real-time chart updates
  - Embedded in final HTML

boxplot-plugin.js (2 KB)
  - Custom Chart.js plugin for box & whisker plots
  - Draws min/q1/median/q3/max whiskers
  - Embedded in final HTML

build.py (11 KB)
  - Build automation script
  - Combines data.json + app.js + boxplot-plugin.js into single HTML
  - Embeds all external library CDNs in HTML headers

================================================================================
BUILD PROCESS
================================================================================

Step 1: prep_data.py
  Input: target_pdp_results.xlsx (2,673 rows from Target.com)
  Output: data.json
  Actions:
    - Parse prices from "$XX.XX" to numeric
    - Split fit descriptions: "X Leg with Y Fit" → leg_shape, fit_style
    - Standardize rise categories
    - Map 500+ color values to 6 wash categories
    - Parse inseam lengths
    - Extract cotton percentages
    - Aggregate brand-level statistics

Step 2: Manually verified
  - app.js created with ES5-only code
  - boxplot-plugin.js created for custom chart rendering
  - Both validated with Node.js syntax checker

Step 3: build.py
  Input: data.json, app.js, boxplot-plugin.js
  Output: jeans_dashboard.html
  Actions:
    - Read all components
    - Embed data as window.DATA JSON object
    - Embed app.js script block
    - Embed boxplot-plugin.js script block
    - Include HTML5 template with all charts
    - Include inline CSS styling
    - Include CDN links for Chart.js, DataLabels, D3, Montserrat

Step 4: Validation
  - Script syntax check with Node.js
  - HTML element ID verification
  - All getElementById calls have matching elements
  - Data integrity check (2,673 rows × 58 brands)
  - Chart initialization verification

All steps automated and tested.

================================================================================
SUPPORT & TROUBLESHOOTING
================================================================================

Dashboard won't load:
  - Ensure browser is modern (Chrome 60+, Firefox 55+, Safari 12+)
  - Try refreshing the page
  - Check browser console for errors (F12 → Console tab)

Charts not showing:
  - Click a brand pill to activate filtering
  - Try toggling price type (Current/Original)
  - Reload the page

Slow performance on mobile:
  - Dashboard is optimized for desktop (1400px+)
  - Mobile works but with smaller chart sizes
  - All functionality is preserved

Offline use:
  - Dashboard works completely offline
  - No internet connection required
  - All data is embedded in HTML file
